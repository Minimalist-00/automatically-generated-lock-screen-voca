# アプリケーション UIデザインの原則

現在のアプリケーションのUIデザインを構成している要素を、デザインの観点から具体的に言語化したものです。新アプリ開発の際の指針として活用します。

## 1. カラーリング（柔らかさと統一感）
*   **パステル＆同系色まとめ**: 背景は極めて淡いミントグリーン、テキストには真っ黒ではなく「深みのある緑（#4A6B65）」を使用しています。コントラストを少し落とすことで、目に優しく、穏やかで安心感のある印象を与えます。
*   **色付きの影（カラーシャドウ）**: カードなどの立体感を出す際、黒い影ではなく「メインカラーと同系色の影（緑がかった影）」を使っています。これにより、全体が濁らず、ふんわりと浮いているような透明感・清潔感を生み出しています。

## 2. 形状（徹底した丸みと親しみやすさ）
*   **完全な角丸（ピル型）**: ボタンの角を限界まで丸め（`border-radius: 9999px`）、カードや入力フォームも大きめの角丸（16px〜24px）に設定しています。画面内から「尖った角」を完全に排除し、触りたくなるようなキュートさと安全性を表現しています。
*   **文字とアイコンの丸み**: UIの枠組みだけでなく、フォント（M PLUS Rounded 1cなどの丸ゴシック）やアイコン（Material Symbols Rounded）に至るまで「丸み」を持つものを選択し、細部までデザイン言語を統一しています。

## 3. インタラクション（心地よい触感）
*   **物理的なフィードバック**: ボタンにカーソルを合わせると少し「フワッ」と浮き上がり、クリックすると少し「沈み込む」ようなマイクロインタラクション（微細なアニメーション）が設定されています。これにより、有機的で愛着の湧く操作感になっています。
*   **優しいフォーカスリング**: 入力欄などを選択したときに表示される枠線を、ブラウザ標準のものから「テーマカラーに合わせた太く柔らかいアウトライン」に変更しており、操作中の状態変化も世界観を崩しません。

## 4. ノイズの排除（アプリらしい没入感）
*   **枠線に頼らない区切り**: 要素の境界線をあえて引かず、背景とのわずかな色の差と「柔らかい影」だけで要素を独立させています。
*   **スクロールバーの非表示**: 画面上のスクロールバーをあえて見えなくすることで、Webページ特有のノイズを消し、ネイティブアプリ（スマホアプリ）のようなスッキリとした没入感のある画面構成にしています。

---
**総括:**
一言で言えば**「尖りを排除した、優しく触り心地の良いふんわりデザイン」**です。人間関係というデリケートなテーマを扱うアプリにおいても、この「安心感」「温かみ」「威圧感のなさ」というデザイン言語は非常に相性が良いはずです。
